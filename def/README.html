This directory is a sparse copy of the def directory hierarchy.

It contains files that are specific to the VM version of the PHR, and are not used for the GitHub version.

Contents are as follows:

  • etc_httpd - a directory that contains configuration files that should be placed in the /etc/httpd directory on the VM.
  • Gemfile - a file used in setting up the Ruby Gems on the VM. It will be removed by the installation process.
  • LICENSE.md - the markdown version of the License that needs to go with the VM.
  • packages - the def/packages directory for the VM. This contains links to the external packages that are included in the VM. The links will appear as broken links on a development machine, but should be valid on the VM. (If they're not, there's a problem.) This directory should be copied to the main def directory on the safe branch before creating a tar file of the safe branch for the VM. (Once you do that, check out the safe branch again to restore the overwritten directory.
  • proto1 - a sparse copy of the def/proto1 directory. This contains files that differ from the majority of the safe branch files. These files should be copied to the proto1 directory of the safe branch before creating a tar file of the safe branch for the VM. (Once you do that, check out the safe branch again to restore the overwritten files).
  • README.html - this file.
