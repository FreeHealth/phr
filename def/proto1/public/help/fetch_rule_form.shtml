Fetch Rules

Fetch rules derive their result in two steps:

  1. a set of items, or records, is retrieved from the database, using selection criteria that you specify to define which items should be included in the set; and
  2. the items are compared to comparison criteria that you specify, and the rule value is derived from the results of the comparison.

The rule may be assigned one of three values:

  1. no value, which is assigned when no items are retrieved in step 1;
  2. true, which is assigned when one or more items are retrieved in step 1 and the comparison criteria specified in step 2 are met; or
  3. false, which is assigned when one or more items are retrieved in step 1 but the comparison criteria specified in step 2 are not met.

The result of the rule is passed to the rule action that you specify. An affected field must be specified for each action, and is the field that gets the results of the action. For example, a hide action will hide the affected field if the rule evaluates to true, or show it if the rule evalutes to false. Some rule actions require parameters, while others do not. See the rule actions section for more information.
